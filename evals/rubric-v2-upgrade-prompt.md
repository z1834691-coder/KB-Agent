## 修改需求 A：用户指定（必须完整落实）

### U1 · D1 时效性 × 经典性分型
- 建立知识类型分类法，建议四类：`奠基经典`（如 Attention Is All You Need、Hinton 蒸馏）/ `框架方法论`（选型框架、评测方法）/ `产品版本与数据`（模型版本、价格、榜单）/ `新闻动态`
- 每类定义各自的保鲜策略：**奠基经典不考核时效**，改考核"历史定位标注完整性"（开创了什么、被什么发展或取代）；框架方法论慢衰减（年检）；产品数据快衰减（月检）；新闻动态极快（周检 + 滚动归档）
- 时效性评分 = 对照该条目**所属类型的保鲜策略**打分，废除一刀切"越新越好"

### U2 · D2 选择眼光客观化
- **a.** 从黄金研读报告提炼"选择眼光的本质规律"清单（每条注明来自哪个标杆的什么证据），作为可复用判据编号写入 rubric 附录；Judge 打分时必须引用判据编号
- **b.** 拆成客观子指标：`信息密度`（单位篇幅的事实/洞察量）、`前沿性`（对照类型保鲜策略）、`洞察深度`（本质规律/一手判断 vs 情绪噪音/正确的废话）
- **c.** 新增一级子指标"**知识分层分类型**"：知识库是否按知识类型（学术论文/官方文档/经验认知/数据榜单/新闻）分类管理，且每类有**定制化选择标准**（论文看引用与复现、经验认知看作者实战背书、数据看口径与样本、新闻看一手性）

## 修改需求 B：专家审查升级项（Claude 提出，用户已阅）

- **C1 · D6 重构 + 锚点去口号化**：把黄金研读 checklist 分配进 D1–D5，作为各维度 10 分锚点的具体化；全文**禁止以人名/产品名作锚点**（如"Simon Willison 级"）。D6 改为"标杆实践采纳率"（达成数/总数），权重降至 5%，释放的 5% 用于 C4 效率约束的配套报告维度或并入 D2（由你论证决定）
- **C2 · 子指标独立计分 + 显式合成**：每个子指标独立 1–10，维度分 = 子指标均值（如加权须注明），杜绝 Judge 子指标间自由裁量
- **C3 · 行为评测 B1 加"可辩护性"通道**：选择/取舍类任务不设唯一正确答案——与参考行为一致直接得分；不一致时 Judge 改评"理由质量"（证据、与 SOP 判据一致性、风险意识），理由充分可得 7+。防止 agent 学会"猜参考答案"
- **C4 · 运行效率约束（不入权重的一票项）**：每轮报告 token 消耗、运行时长、**用户把关耗时**（确认请求数 × 平均处理时间）；超出 v2 中定义的预算 = 该轮直接不合格。理由：eval 不含成本，agent 会用无限算力刷分
- **C5 · 防过拟合机制**（衔接 v0.4 已有 split，把使用纪律写进 rubric 正文）：
  - split 使用纪律：smoke（15）每轮必跑；regression（15，含历史翻车条目）每轮必跑；**holdout（10）每 3 个 agent 版本测一次，结果绝不用于 prompt 调优**；full-only（60）只在里程碑（如 Phase 完成）全量跑
  - 答案隔离纪律：被测 agent 只见 actor-pack，gold 只给 Judge；任何把 gold 内容带进 actor 上下文的运行判无效
  - 每轮行为评测追加 3–5 条当周早报新鲜条目（agent 从未见过）作活体测试，得分单列
  - 训练侧（smoke/regression）与 holdout 分差 > 1.0 = 过拟合警报（dashboard 已支持）
- **C6 · Judge 校准集**：从挑战集选 5–10 条，由用户确认"标准分"；每轮 Judge 先打校准集，平均偏差 > 1.0 → 本轮全部评分标记"低置信"需复评
- **C7 · 状态↔行为联动检查**：连续两轮行为分上升而状态分不动 = "无效运行"信号（单条处理好但库没变好），触发归因
- **C8 · 权重给依据 + 分阶段**：给权重分配写可辩护的依据陈述；定义两套权重——**建设期**（现阶段：可维护性、目标对齐权重高）与**稳态期**（信息质量、内容价值权重高），注明切换条件
