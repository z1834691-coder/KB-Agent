# KB-Agent 评测标准 v0.1

> **定位**：本 rubric 是 agent 迭代的方向盘。所有分数 1–10，必须附证据。
> **版本化**：rubric / 挑战数据集 / agent prompt 三者独立编号，每轮评测记录三元组（如 `rubric v0.1 × dataset v0.1 × agent v0.3`），保证分数跨轮可比。

---

## 0. 两种评测模式

| 模式 | 对象 | 频率 | 数据集 |
|------|------|------|--------|
| **状态评测（State Eval）** | 知识库快照本身 | 周度（Doctor 深度诊断时） | 黄金数据集作参照系 |
| **行为评测（Behavior Eval）** | agent 单次运行的处理表现 | 每次运行后（抽测）+ 周度（全量） | 挑战数据集 |

Baseline = 对「AI知识库 V3」做一次状态评测（见 `04-baseline-report-v0.1.md`）。

---

## 1. 状态评测：六大维度

### D1 · 信息质量（权重 25%）
子指标：**1a** 事实准确性 · **1b** 信源权威度与标注完整性（关键信息可点击溯源）· **1c** 时效性（时效文档在保鲜期内）· **1d** 交叉验证（关键数字/争议命题有 ≥2 独立源或标 ⚠️ 待核实）

| 锚点 | 描述 |
|---|---|
| 1–2 | 大量无源信息；存在已知错误未标注 |
| 4 | 多数内容有源，但存在已入库错误或核心时效文档过期 |
| 7 | 全部关键信息有一手源；无已知事实错误；时效文档 ≤2 周新鲜；存疑均有 ⚠️ |
| 10 | Simon Willison 级溯源纪律；随机抽 20 条零错误；时效由自动告警保障 |

### D2 · 内容价值与目标对齐（权重 25%）
子指标：**2a** 选择眼光/信噪比 · **2b** 前沿性 · **2c** 与"中国头部 AI 公司 PM 秋招"目标的直接相关性（目标公司深度、面试可用弹药）· **2d** 覆盖全面性（MECE 无重大空白）

| 锚点 | 描述 |
|---|---|
| 1–2 | 内容与目标脱节，或大量低价值堆积 |
| 4 | 框架全面但与目标错位（如缺目标公司深度板块） |
| 7 | 每篇能回答"服务什么面试/实战场景"；目标公司有专门板块且深度不低于通用内容 |
| 10 | 行业专家盲评认为选材眼光等同头部从业者；可支撑目标公司模拟面试全程 |

### D3 · 结构与组织（权重 15%）
子指标：**3a** MECE 与分层合理性 · **3b** 交叉引用健康（双链可点击、无死链）· **3c** 知识演化可见（时间标注、新旧对照）· **3d** 可导航性（索引/图谱/标签多入口）

| 锚点 | 描述 |
|---|---|
| 1–2 | 无索引，文档间无关联 |
| 4 | 有索引和分层，但引用不可点击、命名不一致、演化仅靠人工叙述 |
| 7 | 全库双链 + 标签；文件名与索引一致；重要知识有演化标注 |
| 10 | 图谱可视化有信息量；任意概念 ≤2 跳可达；演化时间线成为增值内容 |

### D4 · 呈现与体验（权重 10%）
子指标：**4a** 纪要质量与重点突出 · **4b** 形式适配（文/图/表/链接按内容特性选择）· **4c** 单元长度适中 · **4d** 工具特性利用（Obsidian graph/tags/callout）

| 锚点 | 描述 |
|---|---|
| 1–2 | 纯文本墙，无任何视觉分层 |
| 4 | 有表格与标记体系，但存在超长单文档、图片资产缺失、平台特性闲置 |
| 7 | 该图则图、该表则表；单文档 ≤ 一次专注阅读量；Obsidian 特性实际使用 |
| 10 | AI Index 级图表规范；三种阅读深度（速查/正常/深读)均舒适 |

### D5 · 可维护性与流程（权重 15%）
子指标：**5a** 版本控制与可逆 · **5b** 变更可观测（changelog/diff 自动生成）· **5c** SOP 沉淀 · **5d** 自动化程度

| 锚点 | 描述 |
|---|---|
| 1–2 | 无版本控制，变更不可追溯，全靠人工记忆 |
| 4 | 有 git 但 changelog 靠人写；SOP 零散 |
| 7 | 原子 commit + 自动 changelog；核心流程有成文 SOP 且被遵守 |
| 10 | AI News 级自动管线 + 完整回滚能力 + SOP 随事故自动进化 |

### D6 · 对标顶尖（权重 10%）
方法：对用户选定的每个黄金标杆，抽取 3 项"可移植实践"逐项对照打分（已达成/部分/未达成），换算 1–10。
*（待用户选定标杆后，本节展开为具体 checklist。）*

**综合分 = Σ(权重 × 维度分)。短板规则：任一维度 < 4 视为不合格，触发专项修复，不论综合分多高。**

## 2. 行为评测：四维度（对挑战数据集逐条打分）

| 维度             | 权重  | 打分要点                                   |
| -------------- | --- | -------------------------------------- |
| **B1 判断正确性**   | 40% | 收/不收/标注/路由决策与"专家级参考行为"的一致度；踩中陷阱 = ≤3 分 |
| **B2 处理质量**    | 30% | 纪要是否重点突出、归类是否唯一合理、形式选择是否适配             |
| **B3 溯源纪律**    | 20% | 一手源优先、链接完整、转述与事实区分、⚠️ 标注到位             |
| **B4 边界与升级意识** | 10% | 该问用户时问用户、敏感内容克制处理、超出能力时明说而非硬编          |

单条得分 = Σ(权重 × 维度分)；一轮行为评测得分 = 所有条目均值 + 按十大类别的分类分（暴露弱项类别）。

## 3. Judge 协议（防评分失真）

1. **角色分离**：Judge 用 claude-opus-4-8，与 actor（sonnet-5 日常 / opus-4.8 周度）分离；opus 自评自时必须开独立会话、盲评（不带 actor 的推理过程和自我辩护，只给输入 + 产出 + 参考行为）。
2. **证据强制**：每个子指标分数必须附 ≥1 条证据（`文件:行` 或挑战条目 id + 引文）。无证据的分数无效。
3. **锚点先行**：打分前先复述对应锚点描述，再给分，防止分数漂移。
4. **双评取均**：同一评测独立跑两次，任一维度分差 > 1.5 触发第三次仲裁并记录分歧原因。
5. **人类校准**：周报呈现"Judge 分数 + 证据"，用户可覆写任何分数。**覆写差值是最高价值的飞轮数据**（说明 rubric 或 Judge 与用户标准有偏差），单独记录并驱动 rubric 修订。
6. **回归测试**：历史失败条目永久进回归集，每轮必测；回归通过率单独报告。

## 4. 分数门槛与目标

| 里程碑 | 状态评测综合分 | 行为评测均分 |
|--------|--------------|-------------|
| Baseline（2026-07-11） | **4.7** | 未测（agent 尚未运行） |
| Phase 1 大修完成 | ≥ 6.5 | ≥ 7.0 |
| 稳态运行一个月 | ≥ 8.0 | ≥ 8.5，且回归通过率 100% |

## 5. 评测驱动迭代的 loop（每轮固定跑）

```
运行任务 → Judge 评分（附证据）→ 归因（prompt / SOP / 工具 / 数据源 / rubric 本身，五选一为主因）
→ 针对主因做一处改动（一次只动一类变量，保证归因有效）
→ 同题重跑对比 → 输出 diff 报告 → 结论沉淀进 SOP → 版本三元组 +1
```

每轮产出物落盘：`evals/runs/YYYY-MM-DD-runN.md`（分数、证据、归因、改动、对比），作为飞轮数据的一部分。
