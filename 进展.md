## 1. 项目坐标（背景 · 目标 · 边界）

- **用户身份**：复旦广告/营销背景、AI-native，现韶音 AI 创新 PM；目标 **2026 秋招进入国内头部 AI 大模型公司（DeepSeek/MiniMax/阶跃/Moonshot/智谱）或互联网应用端 AI（阿里/字节）当 AI PM**。
- **产品目标**：把一个手动维护、质量参差的静态 md 知识库，变成一个能**自主诊断→更新→评测→自我优化**的活系统；同时该 agent 本身就是一件可展示、可深挖的 AI PM 作品集（AI Harness）。
- **四大职责**：①持续诊断维护（结构/信息/呈现/迭代）②持续收集前沿高价值知识并可观测地耦合入库 ③visualizer 展示维护与更新情况 ④自主择时/择工具/自评测/自优化/沉淀 SOP/收集数据飞轮。
- **硬边界（来自 DECISIONS.md，不可逾越）**：
  - **HITL 安全边界**：无人值守的自动化**只体检、只出报告、绝不自动改知识正文、绝不自动 commit 正文**。写正文必须有人在场审阅。
  - **git 为唯一真相源**，一切状态持久化在文件系统，任何新会话可从文件恢复。
  - **评测不作弊**：actor 只能看输入包，Judge 才看 gold 答案；把答案带进 actor 上下文的运行判无效。
  - **API key 永不进对话/日志/文件/git**（存 macOS 钥匙串）。

## 4. 本轮（Round 1）鲁棒性修复：Doctor 评分诚实化

### 问题诊断（第三次抓出 eval 自欺）
`kb_ops.py::score_doctor()` 用**纯机械结构信号**冒充质量分：
- **D2 内容覆盖** = `5.0 + 覆盖率×4 + 0.8` → 7/7 存在即 9.8，**完全无视** 3 家档案只有别人一半深度；
- **D5 可面试性** = 只要档案里**出现字符串**"AI PM 面试角度"就给 **10.0**——一篇质量减半的档案也算满分；
- **铁证**：同一 commit `6a2b46d`、同一库快照，飞轮里 1 分钟内出现两条 doctor 记录：**6.7 →（脚本被上调后）9.1**（D1 4.8→8.0、D5 4.0→10.0、D6 8.8→10.0）。诚实人审 Doctor v2 = **6.8**。→ 9.1 是被"调"出来的虚高分，这正是 Goodhart：指标一旦成为目标就不再是好指标。

### 修复方案（已落地，见 git diff）
1. **机械分与质量分物理分离**：`score_doctor()` 输出改为**机械结构完整性分（structural-integrity / lint gate）**，报告顶部强制打诚实 banner：*"本分为机械结构分，非质量分；内容深度与可面试性需 LLM/人工盲评。"*
2. **D2 从"存在"改"深度均衡"**：任何档案行数 < 同类中位数 60% → 判为 `under-depth` 并扣分（直接抓住 247 行的档案）。
3. **D5 不再字符串给满分**：仅当档案含结构化面试块（题+一句话答+追问）才计分，且**质量维度封顶 + 标注 `requires_judge`**，不允许自动打 10。
4. **保留人审/LLM judge 为质量分的唯一来源**——机械分只做"能不能进闸"的门，不做"好不好"的判决。

### 修复后
- 见 §7 本轮 judger 评分与 `evals/robustness_audit.py` 输出、飞轮新增记录、git commit。
